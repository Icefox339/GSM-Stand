# GSM Security Stand (Client-Server + GUI)

Учебный программный стенд на Python для моделирования безопасности передачи данных в GSM-сети:

- **Аутентификация MS** через связку **A3/A8** (вычисление `SRES` и `Kc` из `Ki` и `RAND`)
- **Шифрование MS -> BTS** через **A5-like (CTR-подобный)** потоковый шифр
- Архитектура **клиент-сервер**:
  - клиент = мобильная станция (GUI)
  - сервер = подсистема аутентификации GSM

> Внимание: это **демо-модель для учебных целей**, а не точная реализация криптоалгоритмов стандарта GSM.

## Структура

- `gsm_algorithms.py` — ядро A3/A8 и A5-like (CTR-подобный)
- `gsm_server.py` — TCP-сервер аутентификации и приёма шифрованных данных
- `gsm_client_gui.py` — GUI-клиент (Tkinter)
- `test_gsm_algorithms.py` — минимальные тесты

## Запуск

1. Запустить сервер:

```bash
python gsm_server.py
```

2. В другом терминале запустить GUI-клиент:

```bash
python gsm_client_gui.py
```

3. Нажать **Run Auth + Secure Send**.

## Что происходит в протоколе

1. Клиент отправляет `auth_request` с IMSI.
2. Сервер генерирует `RAND`, считает ожидаемые `SRES`/`Kc`, шлёт challenge.
3. Клиент считает `SRES`/`Kc`, отправляет `auth_response`.
4. Сервер сверяет `SRES`, при успехе отправляет `auth_accept`.
5. Клиент шифрует payload через A5-like (CTR-подобный) с `Kc` и `frame`, отправляет `secure_data`.
6. Сервер расшифровывает и подтверждает (`secure_ack`).
